# 结构光pattern⾃动优化：作业要求

参考OpticalSGD (https://openaccess.thecvf.com/content_CVPR_2020/html/Chen_Auto-

Tuning_Structured_Light_by_Optical_Stochastic_Gradient_Descent_CVPR_2020_paper.html)，实现合成环境下的pattern优化闭环。

# 作业简介

本作业复现OpticalSGD的核⼼思想：把结构光投影图案作为优化变量，通过渲染得到相机观测，使⽤decoder估计投影坐标，再根据解码误差更新图案。与论⽂中的真实相机拍摄不同，本作业使⽤虚拟环境渲染完成整个闭环。

学⽣需要⾃⾏实现全部代码，包括渲染器、OpticalSGD优化器、decoder、实验脚本和报告。允许使⽤通⽤数值计算、⾃动微分和渲染⼯具；推荐 PyTorch 和 Mitsuba。允许使⽤code agent (claude code, codex, etc.), 但最终提交必须能说明系统做了什么、为什么这样做、结果是否⽀持结论。

# 必做模块

# 1.合成渲染器

实现⼀个结构光合成渲染器，不要求⾼质量 path tracing，只需能⽀撑复现实验。⾄少包括：

投影仪模型：将可优化图案投射到场景  
相机模型：⽣成相机观测图像  
场景⼏何：包含有深度变化的物体或表⾯   
⼏何真值：提供投影坐标或对应关系真值  
材质与纹理：⽀持不同外观条件  
可视化：保存图案、渲染图、真值和误差图

渲染器必须做基本⾃检，例如常量图案、条纹图案和真值可视化，证明投影⽅向、相机⽅向和对应关系是合理的。

# 2. OpticalSGD 优化器

实现图案优化闭环，⾄少包括：

多张投影图案的参数表⽰  
当前图案的渲染、解码、损失计算和更新   
有限差分梯度和⾃动梯度两种 patterns -> captured images 的梯度计算⽅式  
图案取值约束和稳定化策略  
训练⽇志、损失曲线、指标和中间可视化

频率约束是必做项：优化后的pattern频率必须不超过预设投影仪分辨率的奈奎斯特极限，否则⽆法在真实硬件上可靠显⽰。

# 3.梯度⽅式对⽐

必须实现并⽐较：

有限差分：对投影图案做扰动，根据渲染变化估计梯度，尽量与OpticalSGD的思路对⻬  
⾃动梯度：通过⾃动微分或可微分渲染链路，直接得到损失对图案的梯度

⽐较内容⾄少包括收敛速度、最终解码精度、运⾏时间、梯度稳定性，以及对噪声、采样数或扰动步⻓的敏感性。两种⽅法应尽量使⽤相同场景、初始图案和评价指标。

# 4. ZNCC 与 ZNCC-NN decoder

必须实现论⽂中的 ZNCC decoder 和 ZNCC-NN decoder，并在相同实验设置下⽐较。

ZNCCdecoder应基于局部强度模式和归⼀化相关性进⾏匹配。ZNCC-NNdecoder应在局部匹配特征基础上加⼊可学习模块，⽤于提升鲁棒性。报告中需要⽐较⼆者在精度、稳定性、运⾏时间和复杂材质下的表现。

# 5. 材质与纹理

必须设置不同材质和纹理来测试鲁棒性。⾄少实现⼤理⽯材质或⼤理⽯纹理；建议再加⼊⽊头、磨砂玻璃和简单漫反射基准。报告中需要展⽰不同材质下的渲染图、优化图案和解码误差，并分析哪些外观更容易导致失败。
